## Requirements

All new software installation or software licensing renewal requests require the following three components: (1) IT Review, (2) Procurement Contract Review or requisition purchase, and (3) IT installation request ticket if needed.  Please contact your IT support technician for assistance, consultation, and guidance as necessary. The following request process is for software that is new to the campus. However, software renewals require the same approvals.  If you are requesting software for a staff or faculty computer that is already available to the campus community and need assistance with the installation, please submit an [IT software installation request service ticket](https://cpp.service-now.com/ehelp?id=sc_cat_item&sys_id=55fea6816ff9e100446af35d5d3ee4af) or review the [list of software applications](../software-list.shtml).

**Important:** All existing software that requires a licensing renewal also requires an [ATI/ITR review request](https://cpp.service-now.com/ehelp?id=sc_cat_item&sys_id=bc4674596f0d43007996073aea3ee428) and a [New Contract Request Form Portal](https://app.smartsheet.com/b/form/21ec2e609b8e46c5a0e3ea481ec2da8d).

**Step 1:** All new software or software requiring a renewal agreement requires an IT Review.

* [Submit an ATI/IT review request](https://cpp.service-now.com/ehelp?id=sc_cat_item&sys_id=bc4674596f0d43007996073aea3ee428).

**Step 2:** The software "terms and conditions" and/or End User License Agreements (EULA) require a Contract Request Form (CRF) review by [Procurement](https://www.cpp.edu/procurement/index.shtml).  Even if the software is free, gifted, previously purchased, or open-source (other than GNU/GPL), a Procurement CRF review of the agreement is still required.  The[GNU/GPL v1, v2, v3](https://www.gnu.org/licenses/licenses.html) EULAs have been pre-approved by Procurement.

* [Submit a New Contract Request Form (PDF)](https://www.cpp.edu/procurement/campus/docs/cpp-contract-request-form.rev.12-21.pdf) to [Procurement](https://www.cpp.edu/procurement/campus/forms.shtml).
   Please include the following documentation with your CRF review to Procurement:

  + Software application End User License Agreement (EULA) and/or Terms and Conditions
  + Completed ATI/IT review documentation
* **Note:** If you are purchasing or renewing software licensing, Procurement will review the contractual agreements as a part of the normal requisition process.  Please follow up with [Procurement](https://www.cpp.edu/procurement/campus/purchasing-goods-and-services.shtml) to confirm if a CRF is needed with the requisition and purchase process.

**Step 3:** After both an IT Review and Procurement CRF Review approval, please submit a software installation request as noted below (if needed).

* The college or department is responsible for attaining the vendor's software licensing codes or files and providing IT with the installation files and licensing information.  Please use the it\_licensing@cpp.edu email address when registering for licensing codes, as IT will receive the licensing files and instructions there.
* For installations on staff or faculty computers, please [open a software installation request](https://cpp.service-now.com/ehelp?id=sc_cat_item&sys_id=55fea6816ff9e100446af35d5d3ee4af).
* For installations within a computer lab or classroom, please [open a computer lab/classroom software installation request](https://cpp.service-now.com/ehelp?id=sc_cat_item&sys_id=15c972730f1c4b007a6046ace1050e83&sysparm_category=04c4f1256f61a1000a149e4eae3ee4ea).

**Important:** When submitting a software installation request, please attach or reference your approved ATI/IT Review documentation and approved CRF confirmation from Procurement.
